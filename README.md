# 病历相似度检索系统 - 原理说明

## 核心问题

如何快速判断一份新病历与库中已有病历的相似程度？

## 技术方案

### 1. 文本 → 向量 → 相似度

```
病历文本
    ↓ 特征提取
特征向量 (高维数值数组)
    ↓ 点积/余弦计算
相似度分数 (0~1)
```

### 2. 为什么用向量表示？

- 文本无法直接比较
- 向量化后可做数学运算（距离、相似度）
- 支持高效的最近邻搜索

### 3. 特征提取策略

病历包含多维度信息，每种用不同方式提取：

| 信息类型 | 示例 | 提取方法 | 向量化 |
|---------|------|---------|--------|
| 诊断 | 肺部感染、脑梗死 | 关键词匹配 | HashingVectorizer |
| 检验指标 | 白细胞、CRP数值 | 正则提取数值 | 标准化数值向量 |
| 治疗 | 用药方案 | 关键词匹配 | 词袋向量 |
| 人口学 | 年龄80岁、男 | 正则提取 | 数值编码 |

### 4. HashingVectorizer 的优势

```
固定方案: ["颅脑损伤", "肺部感染", ...] → 新病无法识别 ❌
Hashing: 任意文本 → 固定维度向量 → 新词自动处理 ✅
```

- 2^18 = 262144 维
- 无需训练，任意文本输入
- 新诊断、新药物自动纳入

### 5. 相似度计算

#### 余弦相似度
```
cosine(A, B) = A·B / (|A| × |B|)
```
衡量两个向量方向的相似程度，取值 [0, 1]

#### 距离转相似度
```
similarity = 1 - distance
```

### 6. 加速检索

#### 暴力计算
比较新病例与库中每个病例 → O(N)

#### FAISS 索引
- 将向量划分到多个聚类
- 检索时只比较最近的几个聚类
- 时间复杂度降至 O(log N) 或 O(1)

## 系统架构

```
┌──────────────────────────────────────────────────────────────┐
│                    病历相似度检索系统                          │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐      │
│  │  病例入库    │ -> │  特征提取   │ -> │  向量存储   │      │
│  │  新病历输入  │    │  Hashing   │    │  NumPy数组  │      │
│  └─────────────┘    └─────────────┘    └─────────────┘      │
│                                              │               │
│  ┌─────────────┐    ┌─────────────┐          │               │
│  │  返回相似病例 │ <-  阈值过滤   │ <-  相似度计算            │
│  │  相似度列表  │    │  top-K     │    │  Sklearn    │      │
│  └─────────────┘    └─────────────┘    └─────────────┘      │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 系统流程

```
新病例输入
    │
    ├─ 解析 ──→ 结构化数据 (诊断/检验/用药)
    │
    ├─ 特征提取 ──→ HashingVectorizer + 数值向量
    │
    ├─ 检索 ──→ top-K 最相似病例
    │
    ├─ 阈值过滤 ──→ 保留 ≥ 阈值的结果
    │
    └─ 返回相似病例列表
            │
            └─ 自动入库 (可选)
```

## 核心参数

| 参数 | 说明 | 建议值 |
|-----|------|-------|
| similarity_threshold | 相似度阈值 | 0.6 ~ 0.8 |
| top_k | 检索返回数量 | 10 ~ 50 |
| HASH_DIM | 文本向量维度 | 2^18 = 262144 |

## 使用方式

```python
from retrieval_system import create_system, scan_medical_records

# 创建系统
system = create_system(data_dir='./data', threshold=0.7)

# 扫描文件夹导入病例
records = scan_medical_records('./data/records')
for filename, text in records.items():
    system.search_and_add(text, record_id=filename)

# 或单个病例导入
system.add_record('病例001', 病历文本)

# 检索相似病例
results = system.search(新病例文本, top_k=10)

# 检索+自动入库
results = system.search_and_add(新病例文本, record_id='病例002')
```
