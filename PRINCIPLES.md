# 病历相似度检索系统 - 原理说明

## 核心问题

如何快速判断一份新病历与库中已有病历的相似程度？

## 技术方案

### 1. 文本 → 向量 → 相似度

```
病历文本
    ↓ 特征提取
特征向量 (高维数值数组)
    ↓ 点积/余弦计算
相似度分数 (0~1)
```

### 2. 为什么用向量表示？

- 文本无法直接比较
- 向量化后可做数学运算（距离、相似度）
- 支持高效的最近邻搜索

### 3. 特征提取策略

病历包含多维度信息，每种用不同方式提取：

| 信息类型 | 示例 | 提取方法 | 向量化 |
|---------|------|---------|--------|
| 诊断 | 肺部感染、脑梗死 | 关键词/TF-IDF | 稀疏向量 |
| 检验指标 | 白细胞、CRP数值 | 正则提取数值 | 标准化数值向量 |
| 治疗 | 用药方案 | 药物名称提取 | TF-IDF或词向量 |
| 人口学 | 年龄80岁、男 | 正则提取 | 数值编码 |

### 4. 相似度计算

#### 余弦相似度
```
cosine(A, B) = A·B / (|A| × |B|)
```
衡量两个向量方向的相似程度，取值 [-1, 1]，通常映射到 [0, 1]

#### 加权融合
由于病历有多维特征，最终相似度为加权组合：
```
Similarity = w1×诊断相似度 + w2×检验相似度 + w3×治疗相似度
```

### 5. 加速检索

#### 暴力计算
比较新病例与库中每个病例 → O(N)

#### FAISS 索引
- 将向量划分到多个聚类
- 检索时只比较最近的几个聚类
- 时间复杂度降至 O(log N) 或 O(1)

### 6. 为什么选 FAISS / sklearn？

| 库 | 适用规模 | 优点 |
|---|---------|------|
| sklearn | < 1万 | 简单、无依赖 |
| FAISS | 100万+ | 毫秒级检索、内存优化 |

## 系统流程

```
新病例输入
    │
    ├─ 解析 ──→ 结构化数据 (诊断/检验/用药)
    │
    ├─ 特征提取 ──→ 特征向量
    │
    ├─ 检索 ──→ top-K 最相似病例
    │
    ├─ 阈值过滤 ──→ 保留 ≥ 阈值的结果
    │
    └─ 返回相似病例列表
            │
            └─ 自动入库 (可选)
```

## 核心参数

| 参数 | 说明 | 建议值 |
|-----|------|-------|
| similarity_threshold | 相似度阈值 | 0.6 ~ 0.8 |
| top_k | 检索返回数量 | 10 ~ 50 |
| 特征权重 | 诊断/检验/治疗 | 0.4/0.3/0.3 |

## 扩展方向

1. **医学本体**: 使用 ICD、SNOMED CT 编码提升诊断匹配
2. **深度学习**: 用医学BERT做语义向量化
3. **时序分析**: 检验指标随时间变化的趋势相似度
